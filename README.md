# blind_vertrauen
Blinde Signaturen und ihr Potenzial fürs Web


Blinde Signaturen erlauben es, eine Nachricht signieren zu lassen, ohne
deren Inhalt dem Signierer offenzulegen, und lösen damit den Zielkonflikt
zwischen Authentifizierung und Anonymität. Obwohl das Konzept seit 1983
bekannt ist und die zugrundeliegenden Verfahren inzwischen standardisiert
sind, ist wenig systematisch erfasst, wo blinde Signaturen im Web
tatsächlich eingesetzt werden. Diese Arbeit grenzt blinde Signaturen
zunächst von klassischen digitalen Signaturen ab und arbeitet ihre
Sicherheitsanforderungen und Varianten heraus. Darauf aufbauend wird der
reale Einsatz im Web mit einer systematischen Suche auf den Plattformen
GitHub, GitLab und Codeberg untersucht. Aus 1838 eindeutigen
Treffern verblieben nach automatisierter Filterung und
mehrstufiger manueller Sichtung acht Web-Anwendungen mit im Code
nachvollziehbarem Einsatz blinder Signaturen. Diese verteilen sich auf
vier Kategorien: anonyme Token gegen Bot-Erkennung,
datenschutzfreundliche Betrugsprävention, IP-Schutz im Browser sowie
anonyme KI-Inferenz. Über alle Kategorien hinweg dient die blinde
Signatur demselben Zweck, der Entkopplung eines Berechtigungsnachweises
von der Identität des Berechtigten. Der produktive
Einsatz konzentriert sich bislang auf wenige große Infrastrukturanbieter.
Die Ergebnisse legen nahe, dass die begrenzte Verbreitung nicht am
kryptographischen Verfahren liegt, sondern an den organisatorischen und
systemischen Voraussetzungen seines Einsatzes.
